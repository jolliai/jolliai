Or will it only read the transcript after the last commit was made?

ASSISTANT:
It only reads entries **after the last commit's cursor position**. Look at line 191:

```kotlin
val cursor = SessionTracker.loadCursorForTranscript(session.transcriptPath, cwd)
```

The cursor tracks where the previous commit left off (line number in the JSONL file). After reading, line 236 saves the new cursor position:

```kotlin
SessionTracker.saveCursor(result.newCursor, cwd)
```

So the BP summary generator only sees the **new transcript entries since the last commit**, not the full conversation history. This means:

- First commit in a session: gets everything from session start to that commit
- Subsequent commits: only the new turns since the previous commit

This is fine for accumulation — each commit adds bullets for whatever was discussed since the last one. But it does mean the LLM won't have full conversation context for later commits, just the delta. The existing bullets passed as context help compensate for that.
