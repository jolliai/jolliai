## What requires renaming

If you fork this project and distribute a modified version, you must rename it so it is not confusingly similar to the Marks. This protects users from being misled about the origin and quality of what they are installing.

Concretely:

- **Do not** publish a fork called "Jolli Memory," "Jolli Memory Pro," "Jolli Memory Plus," "Jolli Memory Community Edition," or any name that incorporates "Jolli" in a way that suggests it is the official project or affiliated with Jolli.
- **Do not** publish packages under the `@jolli.ai` npm scope, or publish a VS Code / IntelliJ extension whose display name or marketplace listing uses the Marks.
- **Do not** use the Jolli logo or visual identity for your fork.

You may, of course, note in your fork's README that it is "based on Jolli Memory" or "originally forked from Jolli Memory" — that is factual reference, which is allowed.

If you want to do any of the above, contact us first at **legal@jolli.ai**. We are generally happy to discuss reasonable use; we just need to know about it.

## Why this matters

The Apache 2.0 license keeps the code open. The trademark policy keeps the *name* trustworthy: when someone installs "Jolli Memory" they should know it is the project maintained at this repository. Without this separation, a fork could ship a backdoored version under our name and damage the trust users place in the project. This is a standard pattern — projects like Mozilla, Kubernetes, and Apache itself operate the same way.
